Interpreting and running StagedMockChain

interpretAndRunWith :: (forall m. Monad m => MockChainT m a -> m res) -> StagedMockChain a -> [(res, TraceDescr)] Source #

Interprets the staged mockchain then runs the resulting computation with a custom function. This can be used, for example, to supply a custom InitialDistribution by providing runMockChainTFrom.

type InterpMockChain = MockChainT (WriterT TraceDescr []) Source #

The semantic domain in which StagedMockChain gets interpreted; see the interpret function for more.

interpret :: StagedMockChain a -> InterpMockChain a Source #

The interpret function gives semantics to our traces. One StagedMockChain computation yields a potential list of MockChainT computations, which emit a description of their operation. Recall a MockChainT is a state and except monad composed:

    MockChainT (WriterT TraceDescr []) a
=~= st -> (WriterT TraceDescr []) (Either err (a, st))
=~= st -> [(Either err (a, st) , TraceDescr)]

Human Readable Traces

prettyMockChainOp :: NonEmpty Wallet -> MockChainOp a -> TraceDescr Source #

Generates a TraceDescription for the given operation; we're mostly interested in seeing the transactions that were validated, so many operations have no description.

newtype TraceDescr Source #

A TraceDescr is a list of Doc encoded as a difference list for two reasons (check ShowS if you're confused about how this works, its the same idea). 1) Naturally, these make for efficient concatenation 2) More importantly, this makes it easy to define the empty TraceDescr as TraceDescr id instead of relying on emptyDoc, which generates empty lines when used with vsep. This avoids generating these empty lines
